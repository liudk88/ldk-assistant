# 语音输入功能 详细设计

> v1 核心功能：通过语音实现电脑文字输入，支持电脑麦克风和手机端两种输入方式。

## 1. 功能概述

### 1.1 目标

用户通过语音（或手机文字）输入内容，系统智能判断用户意图：

- **输入模式** — 识别出的文字直接注入到电脑光标位置
- **指令模式** — 识别出的文字交给 Agent 编排层执行

### 1.2 输入方式

| 输入方式 | 描述 | 场景 |
|---------|------|------|
| 电脑麦克风 | 直接对电脑说话 | 人在电脑前，不想打字 |
| 手机按住说话 | 手机打开网页，按住说话按钮 | 手机当无线麦克风 |
| 手机文字输入 | 利用手机输入法自带的语音转文字，生成文字提交 | 利用手机输入法已有的语音能力，省去自己实现 ASR |

### 1.3 整体流程

```
用户（声音/文字）
      │
      ├─ 电脑麦克风（音频流）
      ├─ 手机按住说话（音频流）
      └─ 手机文字输入（文字流）
      │
      ▼
  语音识别（ASR）          ← 仅音频流需要，文字流跳过
      │
      ▼
  意图分析
      │
      ├─ 输入模式 → 文字注入电脑光标位置
      └─ 指令模式 → 交给 Agent 编排层执行
```

## 2. 意图分析

### 2.1 双模式识别

意图分析支持两种方式，按优先级依次判断：

#### 优先级 1：前缀词（明确指定）

| 前缀 | 模式 | 示例 |
|------|------|------|
| "输入" 开头 | 输入模式 | "输入今天天气很好" → 电脑光标处出现"今天天气很好" |
| "指令" 开头 | 指令模式 | "指令打开微信" → 交给编排层执行"打开微信" |

前缀词匹配成功后，**直接确定模式**，不再走 LLM 分析。

前缀词本身不包含在最终内容中：
- "输入今天天气很好" → 实际输入文字为"今天天气很好"
- "指令打开微信" → 实际指令内容为"打开微信"

#### 优先级 2：LLM 智能分析（兜底）

当用户没有使用前缀词时，通过 LLM 分析判断意图。

**判断依据（三条核心）：**

1. **是否要求对方动手执行行动** — 指向系统或他人做事
2. **句式特征** — 常带"帮"、"去"、"拿"、"打开"、"关闭"、"查一下"等行动动词
3. **说话目的** — 要落地结果、安排办事的属于指令；只陈述现状、吐槽感慨、闲聊分享的不属于指令

**极简判断规则：要人办事就是指令，只说事不安排行动就不是指令。**

**注意：** 委婉问句、客气请求本质上仍属于指令（如"能不能帮我打开一下微信"）。

### 2.2 意图分析接口

```typescript
type UserIntent = 'input' | 'command';

interface IntentResult {
  intent: UserIntent;        // 用户意图
  content: string;           // 提取的纯净内容（去除前缀词后）
  confidence: number;        // 置信度 0~1
  method: 'prefix' | 'llm';  // 判断方式
}

interface IntentAnalyzer {
  analyze(input: string): Promise<IntentResult>;
}
```

## 3. 语音输入端（电脑麦克风）

### 3.1 工作流程

```
用户按住快捷键（开始录音）
      │
      ▼
  采集音频流（系统麦克风）
      │
      ▼
  VAD 检测（用户松开快捷键 或 静音超时）
      │
      ▼
  ASR 识别（FunASR）
      │
      ▼
  意图分析
      │
      ├─ 输入模式 → 文字注入电脑光标
      └─ 指令模式 → 编排层执行
```

## 4. 语音输入端（手机端）

### 4.1 手机按住说话

```
手机打开网页（单页应用）
      │
      ▼
  按住"说话"按钮 → 开始录音
      │
      ▼
  音频流实时上传到服务端（WebSocket）
      │
      ▼
  ASR 识别（FunASR）
      │
      ▼
  意图分析
      │
      ├─ 输入模式 → 推送文字到电脑端 → 注入光标
      └─ 指令模式 → 推送到编排层执行
```

### 4.2 手机文字输入

```
手机打开网页（单页应用）
      │
      ▼
  点击文字输入框
      │
      ▼
  用户使用手机输入法（自带语音转文字）
      │
      ▼
  提交文字到服务端（HTTP / WebSocket）
      │
      ▼
  意图分析
      │
      ├─ 输入模式 → 推送文字到电脑端 → 注入光标
      └─ 指令模式 → 推送到编排层执行
```

### 4.3 手机端设计要点

- **单页应用**：一个 HTML 页面，不需要安装 App
- **两种入口**：按住说话按钮 + 文字输入框
- **连接管理**：手机端通过 WebSocket 连接到服务端，需要考虑断线重连
- **状态反馈**：录音状态、识别中的中间结果、最终结果的实时展示

## 6. 技术方案

### 6.1 ASR 方案

| 阶段 | 方案 | 说明 |
|------|------|------|
| v1 | FunASR（本地） | Paraformer 模型，中文效果好，无网络依赖，无费用 |
| v2 预留 | Whisper | 其他语言或更高精度需求时补充 |

FunASR 部署方式：通过 funasr-wss-server 提供 WebSocket 服务，本项目的 ASR 适配器连接该服务。

### 6.2 通信协议

| 链路 | 协议 | 说明 |
|------|------|------|
| 手机 → 服务端 | WebSocket | 音频流实时传输、文字输入推送 |
| 电脑麦克风 → 服务端 | 本地通信 | 本地进程间通信 |
| 服务端 → 电脑客户端 | 本地通信 | 文字注入指令下发 |
| 服务端 → FunASR | WebSocket | 音频流发送 + 识别结果接收 |

### 6.3 手机端页面

- 纯前端单页应用（HTML + CSS + JS）
- 通过 WebSocket 与服务端通信
- 无需打包发布，浏览器直接打开 URL

### 9.3 详细类设计

#### IntentAnalyzer（意图分析器）

**职责：** 接收文字，判断用户意图是输入模式还是指令模式。优先用前缀词匹配，未命中则走 LLM 分析。

```typescript
// intent/types.ts

type UserIntent = 'input' | 'command';

interface IntentResult {
  intent: UserIntent;         // 用户意图
  content: string;            // 纯净内容（去除前缀词后）
  confidence: number;         // 置信度 0~1
  method: 'prefix' | 'llm';  // 判断方式
}

/** 意图分析配置 */
interface IntentAnalyzerConfig {
  /** LLM 模型配置ID（用于 LLM 意图判断） */
  modelConfigId: string;
  /** 前缀词映射 */
  prefixes: {
    input: string[];    // 输入模式前缀，如 ['输入']
    command: string[];  // 指令模式前缀，如 ['指令']
  };
}
```

```typescript
// intent/analyzer.ts

class IntentAnalyzer {
  constructor(config: IntentAnalyzerConfig, llmProvider: LLMProvider)

  /**
   * 分析用户意图
   * @param text - ASR 识别结果 或 手机端提交的文字
   * @returns IntentResult
   *
   * 处理流程：
   * 1. 遍历前缀词列表，匹配成功直接返回（method='prefix'）
   * 2. 前缀词未命中 → 构造 prompt 调用 LLM 分析（method='llm'）
   */
  analyze(text: string): Promise<IntentResult>

  // --- 内部方法 ---

  /** 前缀词匹配，返回匹配结果或 null */
  private matchPrefix(text: string): { intent: UserIntent; content: string } | null

  /** 构造 LLM 意图分析的 system prompt */
  private buildIntentPrompt(): string

  /** 调用 LLM 判断意图，解析响应为 IntentResult */
  private analyzeByLLM(text: string): Promise<IntentResult>
}
```

#### VoiceInputService（语音输入总调度）

**职责：** 统一管理所有语音输入通道（电脑麦克风、手机端），接收音频流或文字流，调度 ASR 和意图分析，将结果分发到对应输出端。

```typescript
// voice-input/types.ts

/** 语音输入结果 */
interface VoiceInputResult {
  text: string;            // 识别/提交的文字
  intent: UserIntent;      // 意图分析结果
  source: 'mic' | 'phone'; // 输入来源
  timestamp: number;       // 时间戳
}

/** 语音输入事件 */
interface VoiceInputEvents {
  /** 中间识别结果（实时反馈给前端） */
  partial: (text: string) => void;
  /** 最终结果 */
  result: (result: VoiceInputResult) => void;
  /** 错误 */
  error: (error: Error) => void;
}

/** 语音输入配置 */
interface VoiceInputServiceConfig {
  asrConfigId: string;             // ASR 模型配置ID
  intentAnalyzerConfig: IntentAnalyzerConfig;
  onResult: (result: VoiceInputResult) => void;  // 结果回调
}
```

```typescript
// voice-input/voice-input-service.ts

class VoiceInputService {
  constructor(
    config: VoiceInputServiceConfig,
    asrAdapter: ASRStreamProvider,
    intentAnalyzer: IntentAnalyzer,
  )

  /**
   * 处理音频流（来自电脑麦克风或手机按住说话）
   * @param stream - 音频流
   * @param source - 输入来源
   *
   * 处理流程：
   * 1. 调用 ASR 流式识别
   * 2. 收集完整的识别文字
   * 3. 调用 IntentAnalyzer 分析意图
   * 4. 回调 onResult
   */
  handleAudioStream(stream: ReadableStream<Buffer>, source: 'mic' | 'phone'): Promise<void>

  /**
   * 处理文字输入（来自手机文字输入框）
   * @param text - 用户提交的文字
   * @param source - 固定为 'phone'
   *
   * 处理流程：
   * 1. 跳过 ASR，直接调用 IntentAnalyzer
   * 2. 回调 onResult
   */
  handleTextInput(text: string, source: 'phone'): Promise<void>

  // --- 内部方法 ---

  /** ASR 流式识别完成后的处理 */
  private onASRComplete(text: string, source: 'mic' | 'phone'): Promise<void>

  /** 意图分析完成后的分发 */
  private dispatchResult(result: VoiceInputResult): void
}
```

#### AudioCapture（电脑麦克风音频采集）

**职责：** 监听系统麦克风，在用户触发时采集音频流并传递给 VoiceInputService。

```typescript
// voice-input/audio-capture.ts

class AudioCapture {
  constructor(format: AudioFormat)

  /**
   * 启动音频采集
   * @returns ReadableStream<Buffer> 音频流
   *
   * v1 预留接口。具体实现依赖操作系统：
   * - Windows: WASAPI / DirectShow
   * - macOS: CoreAudio
   * - Linux: PulseAudio / ALSA
   */
  start(): ReadableStream<Buffer>

  /** 停止音频采集 */
  stop(): void
}
```

#### PhoneServer（手机端 WebSocket 服务）

**职责：** 为手机端提供 WebSocket 连接，接收音频流和文字输入，推送中间识别结果和最终结果。

```typescript
// phone/types.ts

/** 手机端消息（手机 → 服务端） */
type PhoneMessage =
  | { type: 'audio-start'; format: AudioFormat }
  | { type: 'audio-chunk'; data: ArrayBuffer }
  | { type: 'audio-end' }
  | { type: 'text-submit'; text: string };

/** 服务端消息（服务端 → 手机端） */
type ServerToPhoneMessage =
  | { type: 'partial'; text: string }       // ASR 中间结果
  | { type: 'result'; intent: UserIntent; text: string }  // 最终结果
  | { type: 'error'; message: string };
```

```typescript
// phone/phone-server.ts

class PhoneServer {
  constructor(
    wss: WebSocketServer,
    voiceInputService: VoiceInputService,
  )

  /** 启动手机端 WebSocket 服务 */
  start(): void

  /** 停止服务，断开所有连接 */
  stop(): void

  /** 当前连接的客户端数量 */
  get clientCount(): number

  // --- 内部方法 ---

  /** 处理单个客户端连接 */
  private handleConnection(ws: WebSocket): void

  /** 解析手机端消息并路由 */
  private handleMessage(ws: WebSocket, message: PhoneMessage): void

  /** 推送中间结果给手机端 */
  private sendPartial(ws: WebSocket, text: string): void

  /** 推送最终结果给手机端 */
  private sendResult(ws: WebSocket, result: VoiceInputResult): void
}
```
