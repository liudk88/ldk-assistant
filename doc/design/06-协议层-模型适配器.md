# 协议层 — 模型适配器 详细设计

> 协议层核心模块：将异构的 AI 模型能力统一抽象为标准接口。

## 1. 问题分析

### 现状
AI 系统的核心依赖是各种外部模型和服务，它们存在以下差异：

- **接口各异：** OpenAI、讯飞、阿里、Ollama 各自有不同的 API 格式
- **部署方式不同：** 云端 API vs 本地部署
- **能力类型不同：** LLM、Embedding、ASR、TTS、多模态
- **协议不同：** REST、gRPC、WebSocket
- **流式支持不同：** 有的原生支持流式，有的只支持同步

### 目标
上层（编排层）只面向统一接口编程，不关心底层用了哪个模型、哪个供应商、云端还是本地。

---

## 2. 模型类型与能力矩阵

| 能力类型 | 接口方法 | 云端示例 | 本地示例 | v1 状态 |
|---------|---------|---------|---------|---------|
| **LLM** | `chat()` | OpenAI、DeepSeek、讯飞 | Ollama、vLLM | ✅ 实现 |
| **Embedding** | `embed()` | text-embedding-3 | bge-m3 | 🔮 接口预留 |
| **ASR** | `transcribe()` / `transcribeStream()` | 讯飞、阿里 | FunASR、Whisper | ✅ 实现（FunASR） |
| **TTS** | `synthesize()` | 讯飞、Edge TTS | CosyVoice | 🔮 接口预留 |
| **Multimodal** | `infer()` | GPT-4o、Qwen-VL | LLaVA | 🔮 接口预留 |

### 设计决策
- v1 重点实现 LLM（云端+本地）和 ASR（本地 FunASR）
- TTS、Embedding、Multimodal 定义统一接口，不实现，未来增量扩展
- 接口设计考虑所有类型的共性需求（流式、超时、重试、降级）

## 3. 接口设计

### 3.1 基础接口（所有模型类型共用）

```typescript
/**
 * 模型配置
 */
interface ModelConfig {
  id: string;             // 模型配置唯一标识（UUID），不随名字变化
  name: string;           // 模型标识名（如 "gpt-4o", "qwen2.5:7b"），可变
  provider: string;       // 供应商标识（如 "openai", "ollama"）
  baseUrl?: string;       // API 地址（本地模型必填，云端可选）
  apiKey?: string;        // API Key（云端必填，本地可选）
  maxTokens?: number;     // 最大输出 token
  timeout?: number;       // 请求超时（ms）
  retry?: RetryConfig;    // 重试策略
}

/**
 * 重试配置
 */
interface RetryConfig {
  maxAttempts: number;    // 最大重试次数，默认 3
  backoffMs: number;      // 重试间隔基数（ms），默认 1000
}

/**
 * Token 用量统计
 */
interface Usage {
  promptTokens: number;
  completionTokens: number;
  totalTokens: number;
  modelConfigId: string;  // 关联的模型配置ID（ModelConfig.id）
}

/**
 * 模型响应（同步）
 */
interface ModelResponse<T> {
  content: T;             // 响应内容
  usage: Usage;           // Token 用量
  model: string;          // 实际使用的模型名
  modelConfigId: string;  // 关联的模型配置ID（ModelConfig.id）
  duration: number;       // 耗时（ms）
}
```

### 3.2 LLM 接口

```typescript
/**
 * 对话消息
 */
interface ChatMessage {
  role: 'system' | 'user' | 'assistant' | 'tool';
  content: string;
  name?: string;          // 消息发送者（tool 调用时标识工具名）
  toolCallId?: string;    // 关联的 tool call ID
}

/**
 * 工具调用请求
 */
interface ToolCall {
  id: string;
  name: string;
  arguments: string;      // JSON 字符串
}

/**
 * LLM 响应选项
 */
interface ChatOptions {
  temperature?: number;
  topP?: number;
  stop?: string[];
  tools?: ToolDefinition[];
  stream?: boolean;       // 是否流式输出
}

/**
 * LLM 响应
 */
interface ChatResponse extends ModelResponse<string> {
  toolCalls?: ToolCall[]; // 模型请求的工具调用
}

/**
 * LLM 流式 chunk
 */
interface ChatStreamChunk {
  content: string;
  toolCalls?: ToolCall[];
  finishReason?: 'stop' | 'tool_calls' | 'length';
  usage?: Usage;          // 最后一个 chunk 包含完整 usage
}

/**
 * LLM 提供者接口
 */
interface LLMProvider {
  /** 同步对话 */
  chat(messages: ChatMessage[], options?: ChatOptions): Promise<ChatResponse>;

  /** 流式对话 */
  chatStream(messages: ChatMessage[], options?: ChatOptions): AsyncIterable<ChatStreamChunk>;
}
```

### 3.3 ASR 接口

#### 3.3.1 音频参数

```typescript
interface AudioFormat {
  encoding: 'pcm' | 'wav' | 'mp3' | 'ogg' | 'opus';
  sampleRate: number;       // 采样率，如 16000、44100
  channels: 1 | 2;          // 声道数
  bitDepth?: number;        // 位深，PCM 时适用（16、24、32）
}
```

#### 3.3.2 ASR 配置

```typescript
interface ASRConfig extends ModelConfig {
  id: string;               // UUID，稳定标识
  language?: string;        // 语言代码（如 'zh-CN'），不指定则自动检测
  enablePunctuation?: boolean;  // 是否自动添加标点，默认 true
  enableInverseTextNormalization?: boolean;  // ITN，如"一百二十三"→"123"，默认 true
  vad?: VADConfig;          // 语音活动检测配置
}

/**
 * VAD（语音活动检测）配置
 * 用于判断用户是否在说话、何时说完
 */
interface VADConfig {
  enabled: boolean;         // 是否启用，默认 true
  silenceTimeoutMs: number; // 静音多久判定说完（ms），默认 1500
  minSpeechDurationMs?: number;  // 最短语音时长（ms），过滤噪音，默认 300
  maxSpeechDurationMs?: number;  // 最长单次语音时长（ms），超时强制断句，默认 60000
}
```

#### 3.3.3 ASR 同步接口

```typescript
/**
 * ASR 同步识别（一次性识别，适合短语音）
 */
interface ASRProvider {
  transcribe(audio: Buffer, format: AudioFormat, config?: ASRConfig): Promise<ASRResponse>;
}
```

#### 3.3.4 ASR 流式接口

```typescript
/**
 * ASR 流式 chunk
 */
interface ASRStreamChunk {
  text: string;             // 本次识别的文本（增量，非全量）
  textNormalized?: string;  // ITN 规范化后的文本
  isFinal: boolean;         // 是否为最终结果（一句话识别完成）
  isEnd: boolean;           // 是否为整个流的结束
  speechStartTimeMs?: number;  // 本次语音片段的起始时间
  speechEndTimeMs?: number;    // 本次语音片段的结束时间
}

/**
 * ASR 流式识别（边说边出字，适合长语音/实时输入）
 * 上层分片发送音频数据，服务端实时返回识别结果
 */
interface ASRStreamProvider {
  transcribeStream(format: AudioFormat, config?: ASRConfig): ASRStreamSession;
}

interface ASRStreamSession {
  /** 发送音频分片 */
  send(audio: Buffer): void;
  /** 接收识别结果（流式推送） */
  onChunk(chunk: ASRStreamChunk): void;
  /** 结束音频流（触发最后一段识别） */
  end(): Promise<void>;
  /** 中途取消 */
  abort(): void;
}
```

#### 3.3.5 ASR 响应

```typescript
/**
 * ASR 同步响应
 */
interface ASRResponse extends ModelResponse<string> {
  text: string;             // 识别结果文本
  textNormalized?: string;  // ITN 规范化后的文本
  segments?: ASRSegment[];  // 分句结果
}

/**
 * 语音分句
 */
interface ASRSegment {
  text: string;
  textNormalized?: string;
  startTimeMs: number;
  endTimeMs: number;
}
```

#### 3.3.6 ASR 适配器设计

```
              ┌──────────────┐
              │   业务层      │  ← 音频采集 + 文字输出
              └──────┬───────┘
                     │
         ┌───────────┼───────────┐
         │           │           │
┌────────┴──┐  ┌─────┴────┐  ┌──┴──────────┐
│讯飞 ASR   │  │阿里 ASR  │  │未来 ASR     │
│Adapter    │  │Adapter   │  │Adapter      │
│(实时转写) │  │(实时转写) │  │(...)        │
└───────────┘  └──────────┘  └─────────────┘
```

**v1 云端 ASR 方案：**
- 讯飞实时语音转写（WebSocket 协议，边说边出字）
- 阿里实时语音识别（WebSocket 协议）

**v2 本地 ASR：**
- Whisper（非实时，整段音频识别）

**设计要点：**
- 云端 ASR 均基于 WebSocket 长连接，天然支持流式
- VAD 逻辑优先在适配器内部实现（利用供应商能力），业务层也可叠加本地 VAD
- 各适配器统一输出 `ASRStreamChunk` 格式，上层无感知差异
- 模型配置通过 `ASRConfig.id`（UUID）关联，支持多 ASR 配置切换

## 4. 适配器架构

```
                    ┌──────────────┐
                    │   编排层     │
                    └──────┬───────┘
                           │ 依赖抽象接口
                    ┌──────┴───────┐
                    │ ModelManager │  ← 模型管理：选择、切换、降级
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
     ┌────────┴──┐  ┌─────┴────┐  ┌────┴─────┐
     │OpenAI     │  │Ollama    │  │ 未来     │
     │Adapter    │  │Adapter   │  │ Adapter  │
     │(云端统一) │  │(本地统一) │  │(...)     │
     └─────┬─────┘  └─────┬────┘  └────┬─────┘
           │              │            │
     DeepSeek        qwen2.5:7b    ...
     讯飞
     阿里
     OpenAI
```

### 关键设计点

**OpenAI 兼容适配器：** DeepSeek、讯飞、阿里等大多兼容 OpenAI API 格式，一个适配器覆盖大部分云端模型，通过配置区分。

**Ollama 适配器：** 本地模型统一通过 Ollama 服务暴露，一个适配器覆盖所有本地模型。

**讯飞 ASR 适配器：** v1 不实现，讯飞实时语音转写接口不稳定，留待 v2 评估。

**阿里 ASR 适配器：** v1 不实现，需要收费，留待 v2 评估。

**FunASR 适配器：** v1 核心 ASR 方案。阿里达摩院开源，Paraformer 模型中文效果好。通过 funasr-wss-server 提供 WebSocket 服务，适配器连接该服务进行流式识别。

**ModelManager（模型管理器）：**
- 根据配置选择使用哪个适配器（LLM 和 ASR 独立配置）
- 运行时切换模型（如从云端切到本地）
- 降级逻辑：主模型不可用 → 按优先级尝试备用模型
- Token 用量统计汇总（LLM）/ 识别时长统计（ASR）

### 降级策略

```yaml
# 配置示例
models:
  chat:
    primary:
      id: "a1b2c3d4-..."    # UUID，稳定标识
      provider: openai
      name: deepseek-chat
    fallback:
      - id: "e5f6a7b8-..."  # UUID
        provider: ollama
        name: qwen2.5:7b
    circuitBreaker:
      failureThreshold: 5    # 连续失败 5 次触发熔断
      resetTimeoutMs: 30000  # 30 秒后半开探测
```
