# 技术路线图

> 整体技术栈 + 按阶段引入计划。

## 核心技术栈

### 开发语言
- **TypeScript** — 前后端统一，异步 I/O 适合 Agent 场景

### 基础设施

| 技术 | 用途 | 引入阶段 |
|------|------|---------|
| **Node.js** | 运行时 | v1 |
| **Fastify / Hono** | Web 框架 | v1 |
| **WebSocket (ws)** | 实时通信（流式对话、语音流） | v1 |
| **SQLite** | 轻量存储（v1）；生产切 PostgreSQL | v1→v2 |
| **Pino** | 结构化日志 | v1 |
| **YAML + Schema 验证** | 配置管理 | v1 |
| **Docker Compose** | 容器化部署 | v1 |
| **Redis + BullMQ** | 缓存 + 异步任务队列 + DAG 调度 | v2 |
| **Qdrant** | 向量存储（Memory / 知识库语义检索） | v2 |

### AI 模型接入

| 能力 | 方案 | 说明 |
|------|------|------|
| **LLM** | OpenAI 兼容接口 | 支持多供应商切换（云端/本地） |
| **ASR** | Whisper（本地）/ 讯飞/阿里（云端） | 先云端，后续加本地 |
| **TTS** | Edge TTS（免费）/ CosyVoice（本地） | 先 Edge TTS 快速验证 |
| **Embedding** | text-embedding-3-small / bge-m3 | 云端 + 本地备选 |

## 阶段计划

### v1 — 语音输入 + 对话闭环

**目标：** 电脑端语音输入文字 → AI 对话 → 语音回复

| 新增技术 | 解决问题 |
|---------|---------|
| Fastify | API 服务框架 |
| WebSocket | 流式对话、语音流传输 |
| SQLite | 会话存储、配置存储 |
| Edge TTS | 快速验证语音回复能力 |
| 云端 ASR API | 语音识别 |
| OpenAI 兼容 SDK | LLM 调用 |
| Pino | 日志 |
| Docker Compose | 部署 |

### v2 — 意图理解 + 记忆 + 知识库 + 多模型

**目标：** 语音指令执行、持久记忆、知识库问答、本地模型支持

| 新增技术 | 解决问题 |
|---------|---------|
| Redis + BullMQ | 异步任务调度、缓存、短期记忆 |
| Qdrant | 向量存储、语义检索、长期记忆 |
| PostgreSQL | 替换 SQLite，支撑生产负载 |
| 本地 ASR（Whisper） | 离线语音识别、隐私场景 |
| 本地 TTS（CosyVoice） | 离线语音合成 |
| 本地 LLM 推理 | 离线/低成本运行 |
| 模型路由层 | 同一任务多模型切换/降级 |
| MCP | 标准化工具协议 |

### v3 — 高级交互

**目标：** 语音操控电脑、记忆遗忘、Prompt 防护

| 新增技术 | 解决问题 |
|---------|---------|
| 屏幕理解（多模态模型） | 识别 UI 元素、支持语音操控 |
| UI 自动化框架 | 模拟鼠标键盘操作 |
| Memory 压缩算法 | 记忆遗忘与重要性评分 |
| Prompt 安全层 | 防注入、权限控制 |

## 项目结构

```
ai-agent/
├── packages/
│   ├── core/          # 通用框架（ReAct / Memory / Planning / Tool）
│   ├── transport/     # 通信层（WebSocket / HTTP / 飞书适配）
│   ├── perception/    # ASR + 意图理解
│   ├── synthesis/     # TTS
│   └── plugins/       # 业务插件
├── apps/
│   ├── api/           # API 服务
│   └── voice/         # 语音服务
├── configs/           # YAML 配置
├── doc/               # 设计文档
└── docker/            # Docker Compose 配置
```
